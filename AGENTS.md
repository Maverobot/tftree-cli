## Development
* Use `rg` instead of `grep` if available.
* Use `fd` instead of `find` if available.
## Guidelines
* Emphasize modularity and maintainability.
* Test-driven development.
* When working with markdown files, do not use two trailing spaces at the end of lines for line breaks. Use a single line break instead.
* ALWAYS USE THE QUESTION TOOL IF YOU NEED TO ASK USER.
* USE THE QUESTION TOOL TO ASK FOR FEEDBACK OR NEXT STEPS WHENEVER THE CURRENT TASK IS COMPLETE OR BLOCKED.

## Progress Tracking
* Commit progress updates regularly to avoid losing session information. Ask user before committing anything.
